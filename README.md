Api
1. Проверка учетных данных (Авторизация)
Название: Проверка логина и пароля через систему колледжа
Описание: Колледж проверяет логин и пароль и возвращает базовую информацию о пользователе.
Метод: POST
Путь: /api/auth/verify
Тело:
“username”: ”student123”,
“password”: “hashpassword”,

2. Получение расписания занятий ученика/преподавателя
Название: Получение актуального расписания
Описание: Колледж возвращает расписание для конкретного пользователя на указанный день
Метод: GET
Путь: /api/auth/schedule
Тело: ?user_id=ppkstudent1&date=2025-05-05

3. Отправка статуса посещения
Название: Отправка данных о посещаемости
Описание: Наша система отправляет информацию о присутствие/отсутствие студента на занятие 
Метод: POST
Путь: /api/attendance/mark
Тело:
“timemark”: “2025-05-05T08:45:00”,
“lesson_id”: “mayW01D05L02”,
“student_id”: “ppkstudent1”,
“status”: “present”,

4. Отправка документации об отсутствие
Название: загрузка документа доказывающее уважительную причину отсутствия
Описание: oотправляется скан документа для подтверждения статуса пропуска
Метод: POST
Путь:  /api/attendance/upload-document
Тело: 
“student_id”: “ppkstudent1”,
“schedule_id”: “ATTmayW01D05L”,
“lesson_id”: “ATTmayW01D05L02”,
“reason_type”: “illness”,
“comment”: “болел ОРВИ”,
Document: “[документ]”

5. Получение отчета о посещаемости
Название: Запрос отчетов по посещаемости
Описание: Система может запросить отчет о посещении занятий за месяц.
Метод: GET
Путь: /api/reports/attendance
Тело: ?user_id=ppkstudent1&date=2025-05-05
Другие варианты:
?group_id=ppkgroup1135&date_from=2025-05-05&date_to=2025-05-06
?course_id=ppkcourse12025&date_from=2025-05-05&date_to=2025-05-06
